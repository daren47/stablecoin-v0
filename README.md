# Experimental T-Bill-Backed Stablecoin System

> **Status:** Research / Experimental
>
> This repository contains an experimental implementation of a Treasury‑backed stablecoin system and associated yield distribution mechanism. It is not production‑deployed, not audited, and is published for research and developer experimentation.

---

## What this repository is

This project explores whether tokenized U.S. Treasury bills can be used as backing for an on‑chain stablecoin, while allowing non‑institutional participants to share in the economic upside created by stablecoin issuance itself.

The motivating question is whether stablecoin issuance can be structured so that its economic upside is broadly accessible, rather than captured solely by intermediaries.

The system consists of:

* A stablecoin backed by T-Bill exposure
* A complementary token representing a claim on system yield
* A "bank" contract that manages minting, redemption, and backing

The goal is not novelty, but clarity: to express a familiar financial structure in code and examine its properties.

## What this repository is *not*

* Not production ready
* Not audited
* Not permissionless in a regulatory sense
* Not a token sale or investment solicitation

This is a research implementation intended to make a specific design legible.

## Quick start (local dev environment)

The repository includes a reproducible local development setup using **Docker**, **Anvil**, and **Hardhat**, with forked mainnet state for realistic integration testing.

### Prerequisites

* Node.js (>=18)
* npm (>=9)
* Docker
* An Ethereum RPC URL (e.g. from Alchemy or Infura)

### Tested with

* Node.js 18.x
* npm 10.x

### Setup

1. Clone the repository

```bash
git clone https://github.com/daren47/stablecoin-v0.git
```

2. Install dependencies

```bash
npm install
```

3. Create a `.env` file:

```bash
cp .env.example .env
```

Copy your RPC URL into it:

```
RPC_URL=your_rpc_url_here
```

4. Build the Docker image:

    *(Note: Docker commands may require sudo depending on your system configuration.)*

```bash
npm run build-docker
```

5. Start Anvil (via Docker, forked mainnet):

```bash
npm run anvil
```

6. In another terminal, run tests or deploy locally:

```bash
npm test
```

This spins up a local Anvil instance forked from mainnet, deploys the full contract system, and runs the integration tests.

## Design overview (brief)

At a high level:

* T-bill exposure is held by a single system contract
* A stablecoin is minted against this backing
* Yield generated by the backing accrues to the system
* A separate token represents a claim on that yield

The design intentionally mirrors traditional financial concepts rather than inventing new primitives.

For a fuller discussion of motivation, assumptions, and tradeoffs, see the whitepaper below.
